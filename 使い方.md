# ComfyUI-RogoAI-NanoBanana 使い方ガイド

**バージョン**: 1.0.0  
**作成日**: 2026-03-09  
**目的**: YouTube動画制作用・全機能リファレンス

---

## 概要

このパッケージは **Claude Code** が作成した ComfyUI 用カスタムノード群です。  
大きく3つの機能グループに分かれています。

| グループ | 機能 | ノード数 |
|---------|------|--------|
| **Group A: Gemini画像生成** | Google Gemini / Imagen 3 でテキスト→画像 | 1ノード |
| **Group B: 4D タイムラプス（ComfyUI内）** | 時間×空間の4次元動画をComfyUI内で完結 | 8ノード |
| **Group C: 4D グリッド方式（高品質）** | GLB→キーフレームグリッド→MP4（最高品質） | 4ノード |

## Group A: Gemini 画像生成ノード

### RogoAI Gemini Image Generator

Google の AI 画像生成サービスを ComfyUI から直接使えるノードです。

**カテゴリ**: `RogoAI`

#### 入力パラメータ

| パラメータ | 説明 | デフォルト |
|-----------|------|---------|
| `prompt` | 生成プロンプト（英語推奨） | — |
| `model` | 使用モデル（下表） | gemini-2.0-flash-exp |
| `aspect_ratio` | アスペクト比 `1:1` / `3:4` / `4:3` / `16:9` / `9:16` | `1:1` |
| `num_images` | 生成枚数（1〜4） | 1 |
| `api_key` | Google AI Studio の API キー | — |
| `save_key` | 保存/保存しない | 保存しない |
| `negative_prompt` | 避けたい要素（Imagen3のみ有効） | — |
| `reference_image` | 参考画像（省略可） | — |
| `reference_mode` | `none` / `style` / `subject` | `none` |
| `safety_filter` | セーフティレベル（Imagen3のみ） | BLOCK_MOST |

#### モデル選択

| モデル名 | 料金 | 品質 |
|---------|------|------|
| `gemini-2.0-flash-exp-image-generation` | **無料枠あり** | 普通〜良好 |
| `imagen-3.0-generate-002` | 有料 | 高品質 |
| `imagen-3.0-fast-generate-001` | 有料（低コスト） | 良好・高速 |

#### API キーのセットアップ

1. [Google AI Studio](https://aistudio.google.com/app/apikey) でキーを取得（`AIzaSy...`）
2. `api_key` 欄に貼り付け
3. `save_key = 保存する` で実行 → 次回から自動読み込み
4. 保存先: `models/gemini_api_key.txt`

#### 接続例（基本）

```
[プロンプト文字列] → [GeminiImageGenerator] → [PreviewImage]
```

#### 接続例（参考画像あり）

```
[LoadImage] ──────────────────────────────────────────┐
                                                       ↓
[プロンプト] → [GeminiImageGenerator] → [SaveImage]
               reference_mode: subject
```

## Group B: 4D タイムラプスノード群（ComfyUI 内バッチ方式）

時間軸（ライフステージ）と空間軸（カメラ角度）を制御して、フレームごとに画像生成AIを呼び出す方式。  
ルリシジミの卵→成虫のような**ライフサイクル動画**の作成に最適。

### ノード一覧

| ノード名 | カテゴリ | 役割 |
|---------|---------|------|
| NB4D Time Planner | RogoAI/4D | 時刻 t に応じたプロンプト生成 |
| NB4D Camera Path | RogoAI/4D | フレーム番号 → カメラ角度計算 |
| NB4D Prompt Composer | RogoAI/4D | プロンプト + カメラ情報を合成 |
| NB4D Consistency Engine | RogoAI/4D | 参照画像管理（一貫性維持） |
| NB4D Video Assembler | RogoAI/4D | 連番画像 → MP4 |
| NB4D Depth Estimator | RogoAI/4D | 画像から Depth Map を生成 |
| NB4D Parallax Renderer | RogoAI/4D | Depth Map でパララックスワープ |
| NB4D Stage Anchor Manager | RogoAI/4D | 態ごとの実写参照画像を管理 |

---

### NB4D Time Planner（時間軸制御）

t=0.0（卵）〜t=1.0（成虫）の時間軸で、適切なプロンプトを自動的に補間して出力します。

**入力**

| パラメータ | 説明 |
|-----------|------|
| `t` | 時刻（0.0〜1.0）スライダーで手動制御 |
| `frame_index` | バッチのフレーム番号（Primitive Node から接続） |
| `total_frames` | 総フレーム数（Primitive Node から接続） |
| `stages_json` | ライフステージ定義 JSON（デフォルト: ルリシジミ7態） |

**出力**: `stage_prompt`（テキスト）/ `stage_name` / `t_out`（FLOAT）

**デフォルトのステージ定義（ルリシジミ）**

```
t=0.00: egg（卵）
t=0.15: larva_early（1齢幼虫）
t=0.35: larva_late（終齢幼虫）
t=0.50: pupa（蛹）
t=0.65: eclosion（羽化）
t=0.80: adult_open（成虫・翅開）
t=1.00: flying（飛翔）
```

> **ヒント**: `stages_json` を書き換えれば、カブトムシや植物の成長など  
> どんなライフサイクルにも応用できます。

---

### NB4D Camera Path（空間軸制御）

フレーム番号からカメラの水平角（theta）・垂直角（phi）を計算します。

**入力**

| パラメータ | 説明 | デフォルト |
|-----------|------|---------|
| `frame_index` | フレーム番号 | — |
| `total_frames` | 総フレーム数 | 100 |
| `path_type` | `spiral_ascend` / `orbit_fixed` / `static` | spiral_ascend |
| `revolutions` | 周回数 | 1.0 |
| `start_elevation` | 開始仰角（度） | 0.0 |
| `end_elevation` | 終了仰角（度） | 45.0 |

**出力**: `theta`（水平角）/ `phi`（垂直角）/ `distance_desc`

---

### NB4D Prompt Composer（プロンプト合成）

TimePlanner と CameraPath の出力を合成して、最終プロンプトを作成します。

**入力**

| パラメータ | 説明 |
|-----------|------|
| `stage_prompt` | TimePlanner からの接続 |
| `theta` / `phi` | CameraPath からの接続 |
| `distance_desc` | CameraPath からの接続 |
| `subject_name` | 被写体名（例: `butterfly`） |
| `style_nodes` | スタイル指定（例: `photorealistic, 8k, nature documentary`） |

**出力**: `final_prompt`（画像生成ノードへ接続）

---

### NB4D Consistency Engine（一貫性維持）

フレーム間の見た目の一貫性を保つための参照画像管理ノード。

**入力**

| パラメータ | 説明 |
|-----------|------|
| `strategy` | `anchor_based` / `sequential` / `hybrid` |
| `frame_index` | フレーム番号 |
| `total_frames` | 総フレーム数 |
| `anchor_images` | ライフステージごとの基準画像バッチ |
| `previous_frame` | 1コマ前の生成結果 |

**出力**: `ref_image_1`（直前フレーム）/ `ref_image_2`（アンカー）/ `ref_image_3`（固定基準）

---

### NB4D Video Assembler（動画生成）

ComfyUI から出力された連番画像を ffmpeg で MP4 に変換します。

**入力**

| パラメータ | 説明 |
|-----------|------|
| `images` | IMAGE テンソル（バッチ） |
| `fps` | フレームレート（デフォルト: 30） |
| `filename_prefix` | 出力ファイル名のプレフィックス |

**出力**: `video_path`（保存先パス）  
**保存先**: ComfyUI の `output/` フォルダ

---

### NB4D Depth Estimator（深度推定）

画像から深度マップを生成し、Parallax Renderer に渡します。

**入力**

| パラメータ | 説明 | デフォルト |
|-----------|------|---------|
| `image` | 入力画像 | — |
| `method` | `synthetic` / `midas_small` / `midas_dpt` | midas_small |
| `invert` | 遠近反転 | False |
| `blur_radius` | 深度マップのスムージング | 10 |

**出力**: `depth_image`（表示確認用）/ `depth_mask`（ParallaxRendererへ）

> **注意**: `midas_small` / `midas_dpt` は初回起動時にモデルをダウンロードします。  
> オフライン環境では `synthetic` を使用してください。

---

### NB4D Parallax Renderer（パララックスレンダリング）

1枚の画像 + 深度マップから、カメラが動いているような映像を生成します。  
**3幕構成**で自動的にカメラワークが変化します。

| フェーズ | フレーム範囲 | 動作 |
|---------|------------|------|
| イン | 0〜25% | ズームイン（1.0 → zoom_in） |
| 見せ場 | 25〜75% | 左→右 水平パン |
| アウト | 75〜100% | わずかにズームアウト |

**入力**

| パラメータ | 説明 | デフォルト |
|-----------|------|---------|
| `image` | 入力画像 | — |
| `depth_mask` | DepthEstimator からの MASK | — |
| `frame_index` | フレーム番号 | — |
| `total_frames` | 総フレーム数 | 192 |
| `max_shift_x` | 水平最大シフト（画像幅の割合） | 0.04 |
| `max_shift_y` | 垂直最大シフト | 0.01 |
| `zoom_in` | イン時ズーム倍率 | 1.15 |
| `zoom_out` | アウト時ズーム倍率 | 1.05 |
| `fill_mode` | 端の補完方式 `edge_extend` / `mirror` | edge_extend |

**出力**: `warped_frame`（IMAGE）

---

### NB4D Stage Anchor Manager（態アンカー管理）

実写参照画像（iNaturalist等）フォルダを読み込み、ハルシネーションを防止します。

**入力**

| パラメータ | 説明 |
|-----------|------|
| `stage_index` | 現在の態番号（0始まり） |
| `total_stages` | 総態数 |
| `frame_index` | フレーム番号 |
| `anchor_dir` | 実写参照画像フォルダ（`stage_01.png`〜が入ったフォルダ） |
| `stage_exclusions_json` | 態ごとのネガティブプロンプト |
| `previous_stage_image` | 前態の生成画像（省略可） |

**出力**: `ref_science` / `ref_prev_stage` / `negative_supplement` / `is_stage_start`

## Group C: 4D グリッド方式（GLBベース・最高品質）

3D モデル（GLB）を使った高品質な4Dタイムラプス動画生成方法です。  
**スクリプト群 + ComfyUI ノードのハイブリッド方式**です。

### 全体フロー（ステップ順）

```
STEP 1: 実写素材収集
    ↓
STEP 2: 画像前処理（518×518px、背景除去）
    ↓
STEP 3: 3Dメッシュ生成（Hunyuan3D ComfyUI）
    ↓
STEP 4-a: GLBをComfyUIノードでグリッド化 ← 推奨
    または
STEP 4-b: スクリプトで orbit 動画生成
    ↓
STEP 5: 走査パスで MP4 生成
    ↓
STEP 6: 最終動画結合（ffmpeg）
```

---

### ComfyUI ノード方式（STEP 4〜5 推奨）

| `run_name` | 試行名（空白=タイムスタンプ自動） | — |
| `grid_theta` | 角度分割数（8〜360） | 120 |
| `elev_start` | 最初のステージの仰角（度） | 45.0 |
| `elev_end` | 最後のステージの仰角（度） | 5.0 |
| `cam_dist` | カメラ距離 | 2.5 |

#### make_stage_transitions.py

ステージ間の遷移動画（2秒）を自動生成するスクリプトです。  
SSIM（構造的類似度）で自動的に補間アルゴリズムを選択します。

**スクリプト内設定（冒頭の定数を編集）**

```python
STAGES_DIR = Path(r"C:\Users\...\9stages")   # stage_XX.png のディレクトリ
OUTPUT_DIR  = Path(r"D:\NB4D_test\transitions")
ORBIT_DIR   = Path(r"D:\NB4D_test\orbit_mesh")  # orbit動画ディレクトリ
```

**実行**

```powershell
& $PY "...\make_stage_transitions.py"
```

**補間アルゴリズム（自動選択）**

| SSIM | 条件 | 手法 |
|------|------|------|
| 0.55以上 | 変化小（幼虫の成長） | 双方向光学フローワープのみ |
| 0.55未満 | 変化大（卵→幼虫など） | フロー60% + クロスディゾルブ40% |

---

#### 最終動画結合（ffmpeg）

`make_stage_transitions.py` が生成した `concat_list.txt` を使って結合します。

```powershell
& "C:\Users\fareg\AppData\Local\FFmpeg\bin\ffmpeg.exe" -y `
```

## 4D グリッド方式 完全チュートリアル（ルリシジミ9態）

### STEP 1: 素材収集

各ステージの実写写真を以下のサイトで入手します。

| サイト | 特徴 |
|-------|------|
| [iNaturalist](https://www.inaturalist.org) | CC BY-NC、GPS付き観察記録 |
| [BioLib](https://www.biolib.cz) | ヨーロッパ中心の連続写真 |
| [Wikimedia Commons](https://commons.wikimedia.org) | 高解像度・著作権明確 |

検索キーワード: `"Celastrina argiolus" egg larva pupa adult`

### STEP 2: 前処理

各画像を **518×518px** にリサイズして背景を除去します。

- ComfyUI: `ImageRemoveBackground+` ノード（rembg）
- Web: https://remove.bg

フォルダ構成:
```
9stages/
  stage_01.png  ← 卵
  stage_02.png  ← 1齢幼虫
  ...
  stage_09.png  ← 成虫
```

### STEP 3: 3Dメッシュ生成

**別の ComfyUI**（Hunyuan3D 環境）で各ステージを3Dに変換します。

```powershell
cd D:\Python_VENV\for_comfy_3d_wrapp_250507\ComfyUI
./venv/scripts/activate.ps1
python main.py
```

## トラブルシューティング

### `pyrender` がインストールできない

```powershell
pip install pyrender trimesh
```
GPU環境では `pip install pyopengl` も必要な場合があります。

### custom_rasterizer DLL エラー（Hunyuan3D）

```
ImportError: DLL load failed while importing custom_rasterizer_kernel
```

`mesh_render.py` に以下を追加:
```python
import os, torch as _torch
_torch_lib = os.path.join(os.path.dirname(_torch.__file__), 'lib')
if os.path.isdir(_torch_lib):
    os.add_dll_directory(_torch_lib)
```

### カメラが動かずに被写体が動いて見える

`render_orbit_from_mesh.py` 内で `scene.set_pose(cam_node, cam_pose)` が  
`cam_node` を使っているか確認（最初のノード=メッシュを指定していないか確認）。

### 遷移でワープ歪みが激しい

`make_stage_transitions.py` 内の SSIM 閾値を `0.55` から `0.80` に変更して、  
常にディゾルブ混合を使うようにする。

### ffmpeg が見つからない

```powershell
# ffmpeg のパスを確認
where.exe ffmpeg

# なければ winget でインストール
winget install Gyan.FFmpeg
```
